# Skill 优化器

## 核心原则 [HIGHEST_PRIORITY]

### 1. 精简法则（官方推荐）

| 删除 | 保留 |
|------|------|
| README/文档文件 | 核心指令 (<500行) |
| 解释性长文本 | 触发条件 |
| 冗余逻辑 | 强制执行规则 |
| 通用最佳实践 | 明确边界 |

### 2. 触发条件（必加）

```markdown
## 触发条件

当用户请求包含以下关键词时，加载此skill：
- "[关键词1]": [操作描述]
- "[关键词2]": [操作描述]
```

### 3. 强制格式

```markdown
## 执行规则 [HIGHEST_PRIORITY]

你 MUST：
1. 当请求包含 "[关键词]" 时，执行 [动作]
2. 禁止 [具体行为]
3. 如果 [条件]，则 [动作]
```

### 4. 渐进式披露（官方推荐）

- SKILL.md 主文件：<500行，高层流程
- 详细文档放 references/ 目录
- 使用时再加载（JiT加载）

## 优化步骤

1. READ - 完整读取原skill
2. EXTRACT - 提取核心指令
3. ADD_TRIGGER - 添加触发关键词
4. ADD_FORCE - 添加强制执行格式
5. VALIDATE - 检查是否符合检查清单
6. OUTPUT - 输出优化后的skill

## 输出格式

```markdown
# 优化报告

## 原问题
[描述问题]

## 优化内容
- 删除: [删除内容]
- 保留: [保留核心]
- 新增: [新增规则]

## 验证结果
- [ ] Token < 500
- [ ] 有触发关键词
- [ ] 有强制执行规则
```
